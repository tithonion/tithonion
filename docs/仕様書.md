# Docusaurus特化型デスクトップCMS 仕様書

## 1. プロダクト概要
**コンセプト:**
開発環境に依存せず、非エンジニアでも直感的にDocusaurusのコンテンツを管理できるハイブリッド・デスクトップCMS。

**コアバリュー:**
* **環境構築の排除:** PC環境を汚さず、Tauri (Rust) 側で必要なツール（Bun, Git等）の判定とインストール誘導を完結させる。
* **コードベースの設定:** `.tithonion` ディレクトリによるプロジェクト固有設定、カスタムタスクの共有。
* **複数モードエディタ(MYSIWYG、Code):** Docusaurus本物のプレビュー画面を直接触るインライン編集、生コード編集をシームレスに行き来できるエディタ。

---

## 2. システムアーキテクチャ・技術スタック

| コンポーネント | 採用技術 | 役割 |
| :--- | :--- | :--- |
| **Backend (Core)** | **Tauri (Rust)** | 軽量バイナリ。高速なファイルI/O、画像処理（WebP変換等）、シェルコマンドの動的実行など。 |
| **Local Server** | **Bunなど、プロジェクト依存** | プロジェクトの設定に応じて動的に判定、いずれの場合でも共通したインタフェース。 |
| **Frontend (UI)** | **React / TypeScript** | UI構築、複雑な状態管理（エディタ状態、フロントマターのフォーム連携）など。 |
| **Editor Engine** | **CodeMirror 6** | マルチモード編集のコア。カスタムDOMの注入（Decoration）によるプレビュー表現など。 |
| **Data Handling** | **gray-matter** | MarkdownとYAMLフロントマターの相互変換。 |

## 3. コア機能仕様

### 3.1. マルチモード・エディタ (2 Mode Toggle)
用途に合わせて以下のモードを切り替える。
1.  **ソースモード (Source Mode):**
    * 全画面での生Markdown編集。Docusaurus固有構文（`:::note`, MDX等）のシンタックスハイライト。
2.  **プレビュー ＋ インライン編集:**
    * バックグラウンドで プレビューサーバを起動し、Tauri内の `Iframe` に実際のサイトをレンダリング。プレビュー上のテキストをクリックし、オーバーレイ表示されるエディタから直接編集・保存（双方向同期）が可能(document.designModeとobsidianを混ぜたようないエージ)。

### 3.2. スマート・アセット管理
- **Drag & Drop処理:** 画像をエディタにドロップした際、Rust側で自動的にWebP形式に変換(オリジナル保存形式に変更可能)。
- **自動配置:** 指定されたディレクトリ（編集ファイルと同階層）にコピー。
- **パス挿入:** Markdownへ正しい相対パスを自動挿入。

### 3.3. コンテンツ管理・オートメーション
- **テンプレート機能:** 新規作成時、`.tithonion/templates/` 内のMarkdown雛形（変数展開対応）を選択可能。
- **ディレクトリ自動構築:** Docusaurusブログ向けに `YYYY-MM-DD-title/index.md` 構造を自動生成。
- **高速全文検索:** Rustのファイルシステム走査による、プロジェクト内Markdownのインクリメンタル検索。

## 4. プロジェクト設定・拡張仕様 (`.tithonion/`)

プロジェクトごとに `.tithonion/` ディレクトリを配置し、環境・タスクをコードで管理する。

### 4.1. `.tithonion/config.json` (共有設定)
リポジトリにコミットし、チーム全体で共有する。
- 必須依存ツールとバージョン（例: `bun: ">=1.0.0"`, `gh: "latest"`）。
- アセット保存先のルール、WebP自動変換の有効化フラグ。

### 4.2. `.tithonion/tasks.json` (カスタムタスク)
UI上にショートカットボタンとして展開される、プロジェクト固有のシェルコマンド。
- *例 (同期):* `"sync": "git pull origin main --no-rebase"`
- *例 (公開):* `"publish": "gh pr create --fill && gh pr merge --auto --squash"`
- *例 (ビルド):* `"build": "bun run build"`

### 4.3. `.tithonion/schema.json` (スキーマ定義)
- フロントマターの必須フィールドなど。

### 4.4. `.tithonion/local.json` (個人設定)
`.gitignore` 対象。個人の開発環境の差異を吸収する。
- 各ツールのローカル絶対パス。
- エディタのテーマ設定、認証情報など。

## 5. UI / 画面構成

1.  **セットアップウィザード & プロジェクト選択:**
    - 起動時に `lock` ファイル（`bun.lock`等）や必須ツールを判定。
    - 不足ツールがあれば、Scoop / Winget やポータブル版でのインストールをGUIから促す。
2.  **ダッシュボード:** プロジェクト統計（記事数・下書き状況）、最新アクティビティなど。
3.  **メインエディタ:** エディタモードによって画面構成が若干異なる。
4.  **フロントマター:** YAMLデータを直感的なGUIフォームで編集可能にする。
5.  **タグ管理画面:** プロジェクト全体のタグ一覧と正規化。
6.  **メンバー管理画面:** `authors.yml` と同期し、著者情報をUI上で編集。記事作成時にドロップダウンで選択可能にする。
